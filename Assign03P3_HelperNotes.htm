Assignment 3 Part 3 Helper Notes
Helper Note 3P3.1
Experience with past students tells me that correctly understanding the meanings (semantics) of various expressions involving pointers is a common difficulty.
Below are some notes you may find helpful in that regard.
NOTE: You may not see/need all of these in this assignment.
If a function receives an intPtr (i.e., the address of a word of memory on the stack), then
*intPtr = 0 means to put 0 in the word of memory
(need to do sw using the address)
i < *intPtr means to compare the value of i with the value currently contained in the word of memory
(need to do lw using the address)
++(*intPtr) means to make the value contained in the word of memory 1 more than what it currently is
(has the same meaning as *intPtr = *intPtr + 1)
(need to do lw using the address, add 1 to the loaded value, then do sw using the address)
(see how it's used in PopulateArray: how ++(*usedPtr) is done)
--(*intPtr) means to make the value contained in the word of memory 1 less than what it currently is
(has the same meaning as *intPtr = *intPtr - 1)
(need to do lw using the address, add -1 to the loaded value, then do sw using the address)
a2[*intPtr] = a1[i] means to copy the value currently contained in the "element in array a1 whose index is the value of i" into the "element in array a2 whose index is the value currently contained in the word of memory"
(need to do lw to obtain a1[i], do another lw using the address to obtain *intPtr, then do sw to put a1[i] into a2[*intPtr])
(for this one, usedPtr is the address of another word of memory on the stack the function receives)
*(a2 + *usedPtr) = *intPtr means to copy the value currently contained in the word of memory into the "element in array a2 with an offset that is the value currently contained in the memory location whose address is in usedPtr"
(need to do lw using the address to obtain *intPtr, do another lw using usedPtr to obtain *usedPtr, then do sw to put *intPtr into *(a2 + *usedPtr))
(for this one also, usedPtr is the address of another word of memory on the stack the function receives)
*(a2 + *usedPtr) = *(a1 + *intPtr) means to copy the value currently contained in the "element in array a1 with an offset that is the value currently contained in the word of memory" into the "element in array a2 with an offset that is the value currently contained in the memory location whose address is in usedPtr"
(need to do lw using the address to obtain *intPtr, do another lw using *intPtr (just loaded) to obtain *(a1 + *intPtr), do yet another lw using usedPtr to obtain *usedPtr, then do sw to put *(a1 + *intPtr) into *(a2 + *usedPtr))
*intPtr appearing as an argument in a function call means the value currently contained in the word of memory is to be passed as that argument
(need to do lw using the address to obtain *intPtr)
Note that the function may receive the address through a register (if it comes as 1 of the first 4 arguments) or through the stack (if it comes as 1 of the 5th-and-beyond arguments).
If received through the stack, it will typically need to first be loaded (using lw) into a register to be usable.
